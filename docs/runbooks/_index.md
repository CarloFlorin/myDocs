# Runbooks

Runbooks sind für Incidents und Betriebsaufgaben: kurz, checklistenartig, mit klarer **Verification**.

## Regeln
- Jede Procedure suggeriert **Commands + Expected Result**
- Keine Secrets (Tokens/Passwörter) in Klartext
- Am Ende immer: Verification + Escalation
